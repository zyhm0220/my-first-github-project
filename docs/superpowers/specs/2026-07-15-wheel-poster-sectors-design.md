# 影轮扇形海报与自然停靠设计

**日期：** 2026-07-15

**页面：** `styles/neon-wheel/`

**基线提交：** `12749cd966fdb63b9ecea02caa32c1def8684919`

## 目标

把当前轮盘中的小矩形海报卡片改为铺满扇区的电影海报，让轮盘更像真实可旋转的电影转盘；移除扇区内片名，避免旋转停止后矩形卡片和竖排文字显得歪斜。爆米花中心保持固定，不随轮盘旋转。中奖指针允许停在扇区中心附近，而不是每次精确对准中心线。

## 已确认的视觉行为

- 每个候选电影对应一个完整扇区，海报使用 `object-fit: cover` 覆盖扇区可见范围。
- 扇区使用动态 `clip-path` 裁切；候选为 2–8 部时按数量计算扇区边界，候选为 1 部时覆盖整个圆盘。
- 扇区之间保留细小间隙，让底层霓虹分区色作为分隔线露出。
- 轮盘内不显示电影名称；候选名称继续保留在轮盘的 `aria-label` 中。
- 海报像贴在实体轮盘上一样随扇区旋转，不做强制反向校正；停止后允许自然倾斜。
- 中奖扇区继续使用亮度、饱和度和霓虹描边突出，但不恢复矩形卡片边框。
- 爆米花中心从旋转的 `#wheel` 内移到 `.wheel-stage` 的独立覆盖层，始终静止。

## 几何与停靠规则

### 扇区海报

每个 `.wheel-label` 改为覆盖整个圆盘的裁切层：

1. 根据候选数量计算单个扇区角度 `360 / count`。
2. 为每个索引计算扇区中心角、左右边界角和海报视觉中心。
3. 通过 `clip-path: polygon(center, outerStart, outerEnd)` 裁出扇区；边界各内收少量角度形成分隔缝。
4. 海报使用大于扇区包围盒的方形覆盖层，中心落在扇区径向中点，并按扇区方向旋转，保证裁切范围内没有空白。
5. 海报加载失败时隐藏图片，露出原有霓虹扇区底色。

### 自然停靠

`computeTargetRotation` 增加可选的停靠偏移角：

- 默认偏移为 `0`，保持现有调用兼容。
- 实际旋转时生成 `±min(8°, 扇区角度 × 18%)` 范围内的随机偏移。
- 偏移远小于半个扇区角，确保指针始终落在中奖扇区内部。
- 中奖电影选择、避免连续命中、锁定状态和约 4 秒动画保持不变。

## DOM 与代码改动

- `styles/neon-wheel/index.html`
  - `#wheelLabels` 继续保留稳定 DOM 接口。
  - `.wheel-center` 移到 `#wheel` 之后、`.particles` 之前。
- `styles/neon-wheel/wheel.css`
  - `.wheel-label` 改为全圆裁切层。
  - 海报去除矩形卡片尺寸、圆角、边框和片名样式，改为扇区覆盖样式。
  - 静止中心维持现有霓虹圆形视觉和层级。
- `styles/neon-wheel/wheel.js`
  - 新增扇区点位、海报位置和安全停靠偏移的纯函数。
  - `renderWheel` 不再创建片名 `<span>`，只创建海报。
  - `spinWheel` 把安全偏移传给目标角度计算。
- `tests/`
  - 单元测试覆盖偏移后的指针仍位于中奖扇区。
  - 结构/集成测试覆盖无片名、扇区裁切、海报保留、中心位于旋转层外以及失败回退。

## 验收标准

1. 8 部候选时，8 张海报分别填满 8 个扇区，没有矩形卡片和扇区内片名。
2. “爱情”仅 1 部候选时，海报覆盖整个圆盘，转盘仍可抽取。
3. 普通旋转和再次旋转后，海报与扇区保持绑定，不出现卡片脱位；自然倾斜可接受。
4. 指针落在中奖扇区内部，但多次旋转不要求每次对准扇区中心线。
5. 爆米花中心在旋转前、旋转中和旋转后视觉方向与位置不变。
6. 海报失败回退、中奖高亮、类型筛选、搜索、详情、影评和观影历史均不回归。
7. 1280×720、1440×900 和当前批注视口约 1647×738 均无横向溢出，控制台无 JavaScript 错误。

## 非目标

- 不增加电影数量。
- 不修改经典首页。
- 不修改搜索、详情、影评存储或页面其他区块布局。
- 不要求海报在旋转停止后全部保持屏幕正向。
