# 影轮 CineSpin 桌面转盘页面设计

## 背景

Movie Picker 当前是一个单文件中文静态网页，内置 15 部高分电影，支持名称搜索、随机选择、电影详情、五星评分、个人影评和观影历史。项目计划长期增加多种电影选择视觉方案，因此每种新风格需要拥有独立页面，同时复用电影数据、搜索逻辑和影评存储。

本设计只覆盖第一个新风格：参考视觉素材中左上角的深色霓虹转盘页面。第二个老虎机页面将在独立分支和独立任务中设计、实现。

## 产品定位

- **目标用户：** 希望快速决定今晚看什么、同时喜欢电影氛围和轻度游戏化体验的桌面端用户。
- **核心价值：** 把普通随机选择变成有候选、有过程、有揭晓感的电影转盘体验。
- **品牌：** “影轮 CineSpin”。
- **主标题：** “今晚看什么？让命运替你决定”。
- **MVP 重点：** 桌面端转盘视觉、可验证的抽取动画、现有电影功能重排，以及为后续老虎机页面准备可复用的基础逻辑。

## 范围

### 本任务包含

- 新增独立桌面页面 `styles/neon-wheel/`。
- 从现有 15 部电影中生成转盘候选。
- 按类型筛选候选电影。
- 转盘旋转、减速、命中和结果揭晓动画。
- 保留名称搜索、电影详情、五星评分、影评保存、编辑、删除和观影历史。
- 与现有页面共享 `movie_reviews` 本地存储数据。
- 建立可供未来老虎机页面复用的电影数据和无界面基础逻辑。

### 本任务不包含

- 手机端和平板端适配。
- 扩充电影数量。
- 实时电影数据 API。
- 修改或迁移根目录经典页面。
- 风格选择入口。
- 第二个老虎机页面。
- 登录、后端、数据库、支付或用户系统。
- 框架、包管理器或构建流程。

## 页面结构

页面采用桌面端纵向单页结构：

1. **顶部导航：** 左侧显示“影轮 CineSpin”，右侧提供“命运转盘”“搜索电影”“观影记录”锚点。
2. **转盘首屏：** 左侧为品牌文案、说明和主按钮，右侧为大型霓虹转盘；类型筛选沿首屏底部排列。
3. **搜索条：** 在首屏之后提供独立电影名称输入框和搜索按钮，避免搜索功能削弱首屏转盘焦点。
4. **电影详情与评分区：** 抽中或搜索成功后显示海报、标题、类型、豆瓣评分、简介，以及五星评分和影评输入。
5. **观影历史：** 页面底部用三列卡片显示已保存记录及编辑、删除操作。
6. **页脚：** 显示品牌和简短标语。

桌面验证视口为 1280×720 和 1440×900。页面不应产生横向滚动条。

## 视觉系统

### 色彩

- 页面背景：`#060711`。
- 卡片背景：`#111226`。
- 主紫色：`#7C3AED`。
- 靛蓝色：`#4F46E5`。
- 青色点缀：`#38BDF8`。
- 粉色点缀：`#F472B6`。
- 评分黄色：`#FACC15`。
- 主文字：`#F8FAFC`。
- 次文字：`#94A3B8`。

高强度霓虹只用于转盘、主按钮和命中瞬间。普通卡片、搜索框和次按钮使用低亮度边框，避免整页持续发光造成视觉疲劳。真实电影海报保持原色，不叠加统一紫色滤镜。

### 字体与层级

页面使用本地系统无衬线字体栈，中文优先 `PingFang SC`、`Microsoft YaHei`，英文优先系统中的 Inter 风格无衬线字体。页面不依赖在线字体。

- 主标题使用粗体、大字号和紫蓝至粉色渐变强调。
- 英文眉题使用小字号、大字距和全大写样式。
- 正文使用高对比浅灰色和舒适行距。
- 豆瓣评分使用黄色，但不把第三方评分表现成实时数据。

### 组件

- 主按钮使用紫色到靛蓝色渐变、胶囊形状和有限范围光晕。
- 次按钮使用透明深色背景和紫色轮廓。
- 类型筛选为胶囊标签，当前选中项使用实色紫色背景。
- 搜索框使用深色填充、低亮边框和清晰焦点环。
- 反馈提示使用小面积深紫色提示条，不遮挡主要内容。
- 海报加载失败时显示带胶片或爆米花图标的霓虹占位卡。

## 转盘规则

### 候选生成

- 默认筛选为“全部”。
- “全部”从现有 15 部电影中无重复随机抽取 8 部。
- 提供“全部、剧情、喜剧、科幻、动画、爱情”筛选项。
- 选择某类型后，仅使用包含该类型的电影。
- 某类型不足 8 部时，使用全部匹配电影，不复制电影填满 8 个扇区。
- 进入页面或切换类型时重新生成候选并更新扇区。

### 抽取流程

1. 用户点击“转动命运转盘”。
2. 程序先从当前候选中确定结果，再计算对应的最终旋转角度。
3. 旋转期间锁定转盘按钮、类型筛选和搜索，按钮文案变为“命运转动中…”。
4. 转盘完成加速、巡航和减速，旋转 5 至 7 圈，总时长约 4 秒。
5. 指针停靠位置、扇区高亮、电影名称和实际结果必须一致。
6. 使用约 0.8 秒的短促光晕和少量粒子完成揭晓，不持续闪烁。
7. 页面平滑滚动到电影详情与评分区。

### 再次抽取

- “全部”重新随机生成一组 8 部候选。
- 某类型不足 8 部时，保留全部匹配电影并重新排列。
- 候选池中有两部以上电影时，选择结果时排除刚刚命中的电影；若它不在新候选中，则直接从全部新候选中选择。
- 若候选池仅有一部电影，则允许连续命中该电影。

### 减少动态效果

当系统启用 `prefers-reduced-motion: reduce` 时，保留结果可理解性，但将旋转和滚动动画缩短为快速状态切换，并关闭粒子效果。

## 搜索与结果展示

- 搜索保持现有精确匹配和简单包含式模糊匹配。
- 搜索不触发转盘旋转。
- 匹配成功后更新同一个电影详情区，并平滑滚动到该区域。
- 未找到时在搜索条附近显示内联提示，不清除上一条有效电影详情。
- 抽取结果和搜索结果使用同一个详情渲染入口，避免两套展示逻辑产生差异。

## 评分、影评与观影历史

- 用户可以选择 0 至 5 星并填写影评。
- 评分和影评至少填写一项才能保存；允许只评分或只写影评。
- 保存成功后更新观影历史并清空当前评分和输入框。
- 编辑记录时回填电影、星级和影评，保存后更新原记录。
- 删除记录前需要确认。
- 观影历史按存储数组顺序显示，新记录位于最前。
- 用户输入必须通过 `textContent` 或等价安全方式渲染，不拼接为 HTML。

本地存储键继续使用 `movie_reviews`，每条记录保持以下格式：

```js
{
  movieName: '肖申克的救赎',
  rating: 5,
  review: '影评文字',
  date: '2026-07-15'
}
```

读取存储失败、JSON 非法或结果不是数组时，按空数组处理，不阻断页面其他功能。

## 技术结构

```text
movie-picker/
├─ index.html                         # 现有经典页面，本任务保持不变
├─ shared/
│  ├─ movie-data.js                  # 新风格页面的统一电影数据
│  ├─ movie-core.js                  # 搜索、筛选、候选生成和随机选择
│  └─ review-store.js                # localStorage 兼容读写
└─ styles/
   └─ neon-wheel/
      ├─ index.html                  # 页面语义结构
      ├─ wheel.css                   # 页面样式和动画
      └─ wheel.js                    # DOM 渲染和事件协调
```

所有脚本使用普通浏览器脚本并通过受控全局命名空间暴露接口，不使用 ES Module 导入。这样既不需要构建工具，也能保持通过本地静态服务器运行的简单部署方式。

### 共享层职责

- `movie-data.js` 复制现有 15 部电影目录及 TMDB 海报地址，作为所有新风格页面的统一数据源，不操作 DOM。
- `movie-core.js` 提供纯数据操作：名称搜索、类型过滤、无重复候选生成和结果选择，不操作 DOM 或存储。
- `review-store.js` 只处理 `movie_reviews` 的解析、保存和兼容性校验，不渲染界面。

### 页面层职责

- `wheel.js` 管理当前筛选、候选列表、当前电影、旋转状态和正在编辑的记录索引。
- `wheel.js` 调用共享层获取数据，再通过安全 DOM API 渲染转盘、结果和历史记录。
- `wheel.css` 实现视觉系统、转盘几何、状态样式、焦点样式和减少动态效果规则。

根目录经典页面本次不迁移到共享脚本，因此它会暂时保留一份内置电影目录。未来老虎机页面直接复用 `shared/`；经典页面迁移和消除这份临时数据重复作为独立任务处理。

## 数据流

1. 页面初始化时读取电影目录和影评记录。
2. 默认选择“全部”，生成 8 部候选并渲染转盘。
3. 类型变化时重新过滤、生成候选并更新扇区。
4. 点击抽取后，核心层返回结果，页面层计算动画终点并锁定交互。
5. 动画结束后更新当前电影、渲染详情并滚动到结果区。
6. 搜索成功时跳过转盘动画，直接进入相同的详情渲染流程。
7. 保存、编辑或删除影评时，通过存储层写入数据，再重新渲染历史列表。

## 错误与边界处理

- 当前筛选没有匹配电影时，禁用转盘按钮并显示“该类型暂无电影”。
- 转盘正在旋转时忽略重复点击，避免并发动画和结果覆盖。
- 海报加载失败时替换为占位卡，不隐藏电影文字信息。
- 搜索为空时提示用户输入名称。
- 搜索未找到时保留上次有效结果。
- 存储读取异常时使用空历史；存储写入异常时显示保存失败提示，不假装成功。
- 编辑索引失效时取消编辑状态并重新渲染历史，不覆盖其他记录。

## 可访问性与可用性

- 交互元素使用原生按钮、输入框和文本区域。
- 所有按钮支持键盘触发，并具有可见焦点状态。
- 类型筛选使用可识别的选中状态，不只依赖颜色差异。
- 海报包含电影名称替代文本。
- 动画期间通过按钮文案和禁用状态同步表达进度。
- 文字与背景保持清晰对比，不用霓虹光晕承载正文信息。

## 验收标准

- `styles/neon-wheel/` 页面可通过本地静态服务器打开，且不修改现有首页行为。
- 1280×720 和 1440×900 桌面视口无横向溢出，首屏突出左文案、右转盘。
- 默认候选为 8 部不重复电影；类型候选不足 8 部时显示全部匹配电影。
- 旋转期间相关操作锁定，动画约 4 秒，指针位置与结果一致。
- 再次抽取按已确认规则重建候选；候选池有两部以上时不得连续命中同一电影。
- 抽中后自动进入电影详情与评分区域。
- 搜索支持精确和简单模糊匹配；未找到时不清除上次有效结果。
- 电影详情、海报回退、评分、影评保存、编辑和删除均正常工作。
- 页面刷新后历史记录仍存在，并与现有经典页面共享。
- 用户影评以安全文本渲染。
- 键盘焦点清晰，减少动态效果设置生效。
- 页面完整操作流程无控制台错误。

## 验证方式

- 使用本地静态服务器验证页面入口和共享脚本加载。
- 对搜索、类型过滤、候选去重、结果选择和存储兼容逻辑执行针对性测试。
- 在 1280×720、1440×900 两种桌面视口完成浏览器视觉检查。
- 完成“筛选 → 抽取 → 揭晓 → 评分/影评 → 编辑 → 删除”的端到端冒烟流程。
- 验证空搜索、未找到、无候选、海报失败、非法存储数据和减少动态效果状态。
- 最终确认根目录 `index.html` 未发生变化。

## 后续工作

本页面完成并合并后，从更新后的 `main` 创建 `codex/style-neon-slot-machine` 分支。老虎机页面复用 `shared/` 中的电影目录、搜索、候选和影评存储能力，只实现独立的老虎机视觉与抽取交互。
